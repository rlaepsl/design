# claim-verification — 출처 라벨링 / 환각 차단 하네스 (데모)

AI가 만든 모든 주장(claim)에 **출처 라벨**을 강제하고, 근거 없는 위험한 주장(환각)을
**결정론적 규칙**으로 잡아내는 품질 게이트입니다. LLM 채점이 아니라 규칙 기반이라 매번 같은 판정이 나옵니다.

## 라벨 체계

| 라벨 | 의미 | source_ids |
|---|---|---|
| `관찰` | 자료에서 직접 보이는 것 | 필수 |
| `출처확인` | 외부 출처로 확인된 것 | 필수 |
| `추론` | 관찰 기반 해석 | 필수 |
| `확인 필요` | 근거 부족, 확정 불가 | 선택 |
| `제안` | 방향 제안(사실 아님) | 선택 |

## 검사 항목

1. 필수 키 존재 + 라벨 유효성
2. **source_ids 커버리지** — 근거가 필요한 claim 중 실제 근거가 달린 비율 (임계 0.8)
3. **위험 패턴 탐지** — 공식 브랜드 단정 · 가짜 전화 플레이스홀더 · 출처 없는 QR · 법적 문구 · 근거 없는 날짜/마감
4. `verdict`(pass / pass_with_warnings / fail) + `score`(0~100)

## 실행

```bash
python verify_claims.py --claims sample_claims.jsonl
```

`sample_claims.jsonl` 에는 **의도적으로 위험 항목 2건**을 심어 두었습니다(게이트가 잡는지 확인용):

```
verdict : fail
score   : 80
claims  : 6  (source_coverage=0.8)
  FAIL: missing_source_ids:C-005
  RISK: C-005:official_brand_claim :: 공식 브랜드 색상은 파란색 계열이다.
  RISK: C-006:invented_phone_placeholder :: 문의 전화번호는 02-0000-0000 이다.
```

- `C-005` — 출처 없이 "공식 브랜드 색상"을 단정 → 환각 위험으로 차단
- `C-006` — `02-0000-0000` 같은 가짜 전화 플레이스홀더 탐지

> 실제 파이프라인에서는 이 게이트를 통과하지 못한 산출물이 다음 단계로 넘어가지 못합니다.
> 이 데모는 그 핵심 아이디어를 공개용으로 새로 작성한 것으로, 회사/고객 데이터는 포함하지 않습니다.
