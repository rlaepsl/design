# 출처 라벨링 (claim labeling)

AI가 디자인 분석을 하면, 보이는 사실과 추측이 뒤섞이기 쉽다. 특히 **공식 색상·서체·로고
규정, 전화번호, 날짜, 법적 문구** 같은 항목을 근거 없이 단정하면 실무에서 사고로 이어진다.
그래서 이 프로젝트는 **모든 주장에 출처 라벨을 강제**한다.

## 라벨 체계

| 라벨 | 의미 | 출처(source) |
|---|---|---|
| `관찰` | 첨부 자료에서 직접 보이는 내용 | 필수 |
| `출처확인` | 공식/신뢰 외부 출처로 확인한 내용 | 필수 |
| `추론` | 관찰 기반의 합리적 해석 | 필수 |
| `확인 필요` | 근거 부족 — 사실로 확정 불가 | 선택 |
| `제안` | 디자인 방향 제안 (사실 아님) | 선택 |

핵심 규칙: **근거가 필요한 라벨(관찰/출처확인/추론)인데 출처가 없으면 통과 못 한다.**
불확실한 것은 반드시 `확인 필요`로 내려, "모른다"를 "안다"로 둔갑시키지 않는다.

## 자동으로 잡는 위험 패턴

검증기는 다음을 근거 없이 단정하면 환각 위험으로 표시한다:

- 공식 브랜드 단정 — "공식 색상/서체/로고는 …"
- 가짜 전화 플레이스홀더 — `02-0000-0000` 같은 채움값
- 출처 없는 QR 목적지
- 법적/정책 문구 단정
- 근거 없는 날짜·마감

→ 구현·실행 예시: [`code/claim-verification/`](../code/claim-verification/)

## 왜 결정론적인가

LLM에게 "이거 환각이야?"라고 다시 묻는 대신, **규칙 기반**으로 검사한다.
같은 입력엔 항상 같은 판정이 나오므로, 회귀 테스트와 CI에 그대로 얹을 수 있다.
